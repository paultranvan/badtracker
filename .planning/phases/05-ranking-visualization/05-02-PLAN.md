  <action>
Create `app/(app)/ranking-chart.tsx` as a Stack screen (outside tabs, same pattern as `player/[licence].tsx`).

**Screen structure:**
```
ScrollView
├── Header: title + subtitle (i18n)
├── ChartLegend (tappable discipline toggles)
├── LineChart (react-native-gifted-charts)
│   ├── Multiple discipline lines via dataSet prop
│   ├── Milestone markers via customDataPoint + dataPointLabelComponent
│   └── Pointer tooltip for value on press
└── Loading / Error / Empty states
```

**Implementation details:**

1. **Screen component** (`RankingChartScreen`):
   - Get `licence` from `useSession()` (same as dashboard)
   - Call `useRankingEvolution(licence)` for chart data
   - Manage `visibleDisciplines` state: `Record<Discipline, boolean>` initialized all `true`
   - `toggleDiscipline` function flips visibility for a discipline
   - Loading state: centered ActivityIndicator (same style as dashboard)
   - Error state: error message + retry button (same pattern as dashboard)
   - No data state: show `ranking.noData` message

2. **ChartLegend component** (inline or sub-component):
   - Horizontal row of 3 items, centered
   - Each item: colored dot (12px circle) + discipline label text
   - Tappable via Pressable — toggles visibility
   - When hidden: opacity 0.4 on the legend item
   - Use DISCIPLINE_COLORS from rankingChart.ts
   - Labels from i18n: ranking.simple, ranking.double, ranking.mixte
   - Hint text below legend: ranking.legend (small, gray)

3. **LineChart configuration:**
   - Use `dataSet` prop (NOT `data`/`data2`/`data3`) for clean multi-line support
   - Filter dataSets based on `visibleDisciplines` state — only include visible disciplines
   - Use `useMemo` to recompute filtered datasets when visibility or chartData changes
   - For each visible discipline's dataSet:
     - `data`: array of `{ value: number, label?: string }` from chartData.disciplines[i].points
     - `color`: from DISCIPLINE_COLORS
     - `thickness`: 2
     - `hideDataPoints`: true (hide regular dots — only show milestone markers)
     - For milestone points: set `customDataPoint` to render a small colored circle (6px radius) and `dataPointLabelComponent` to show rank text in a small colored pill badge
     - `dataPointLabelShiftY`: -22 to position label above the line
     - Only show `showDataPoint: true` on milestone points
   - Chart container height: 280
   - X-axis: show date labels (formatDateLabel from rankingChart.ts), rotated if needed
   - Y-axis: CPPH values, format as integer
   - Grid lines: light horizontal rules (#e5e7eb), no vertical grid
   - `spacing`: calculate from data point count to fill chart width (aim for chart to use available width)
   - `initialSpacing`: 20, `endSpacing`: 20
   - `yAxisTextStyle`: { color: '#6b7280', fontSize: 11 }
   - `xAxisLabelTextStyle`: { color: '#6b7280', fontSize: 10 }
   - `noOfSections`: 5 (Y-axis divisions)
   - `maxValue`: chartData.maxValue + 10% padding
   - `rulesType`: 'solid', `rulesColor`: '#f3f4f6'

4. **Pointer/Tooltip configuration:**
   - `pointerConfig` on LineChart:
     - `pointerStripHeight`: 240
     - `pointerStripColor`: '#e5e7eb'
     - `pointerStripWidth`: 1
     - `pointerColor`: '#6b7280'
     - `radius`: 5
     - `pointerLabelWidth`: 100
     - `pointerLabelHeight`: 60
     - `autoAdjustPointerLabelPosition`: true
     - `pointerLabelComponent`: renders a small card (white bg, rounded, shadow) showing CPPH value formatted as "XX.X pts"

5. **Milestone marker rendering:**
   - For each data point in a discipline, check if `isMilestone` is true
   - If milestone: the data point gets `showDataPoint: true` and a `customDataPoint` function that renders a View with the discipline's color as background (small circle, 7px radius)
   - Also gets `dataPointLabelComponent`: a small pill/badge with the rank text (e.g., "P10") in white text on discipline-colored background, 9px font, bold, border-radius 4
   - Non-milestone points: `showDataPoint: false` (no dot rendered)
   - Markers appear for ALL visible disciplines, each color-coded to their line color (this happens naturally since markers are per-dataSet)

6. **Styles** (StyleSheet.create):
   - Follow existing app style patterns: white background, 16px padding, consistent typography
   - Primary blue: #2563eb (matches app theme)
   - Section headers: fontSize 18, fontWeight '600', color '#111'
   - Subtitle: fontSize 14, color '#6b7280'
   - Chart container: marginTop 12, paddingRight 8
   - Legend row: flexDirection 'row', justifyContent 'center', gap 16, marginBottom 12
