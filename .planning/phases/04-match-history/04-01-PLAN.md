**2. Create `src/utils/matchHistory.ts`:**

Export the following types and pure functions:

```typescript
// Types
export interface MatchItem {
  id: string;                    // Generated: `${date}-${index}`
  date?: string;
  opponent?: string;
  opponentLicence?: string;
  opponent2?: string;            // Second opponent (doubles)
  opponent2Licence?: string;
  partner?: string;              // Partner (doubles/mixed)
  partnerLicence?: string;
  score?: string;
  discipline?: 'simple' | 'double' | 'mixte';
  round?: string;
  tournament?: string;
  tournamentDate?: string;
  isWin?: boolean;
  pointsImpact?: number;        // CPPH points gained/lost
  setScores?: string[];          // Individual set scores
  duration?: string;
}

export interface MatchSection {
  title: string;      // Tournament name
  date: string;       // Tournament date for display
  data: MatchItem[];  // Matches in this tournament
}

export interface WinLossStats {
  wins: number;
  losses: number;
  total: number;
  winPercentage: number;  // 0-100
}

export type DisciplineFilter = 'all' | 'simple' | 'double' | 'mixte';
```

**Functions to implement:**

- `toFullMatchItem(raw: Record<string, unknown>, index: number): MatchItem` — Convert raw API result to typed MatchItem. Check fields: Date, Adversaire, AdversaireLicence, Adversaire2, Adversaire2Licence, Partenaire, PartenaireLicence, Score, Epreuve/Competition (fallback), Tour, Discipline (map "S"→"simple", "D"→"double", "M"→"mixte"), Resultat/Result/VD for win/loss (same pattern as dashboard toMatchPreview), Points (parseFloat), Sets (split by space into array), Duree, DateCompetition. Generate id as `${raw.Date || 'unknown'}-${index}`.

- `groupByTournament(matches: MatchItem[]): MatchSection[]` — Group matches by tournament name. Each section: title = tournament name (or "Inconnu" if missing), date = first match's tournamentDate or date. Sort sections by date descending (most recent first). Sort matches within sections by date descending.

- `filterByDiscipline(matches: MatchItem[], discipline: DisciplineFilter): MatchItem[]` — Return all if 'all', otherwise filter on discipline field. Matches with undefined discipline are included in 'all' only.

- `filterBySeason(matches: MatchItem[], season: string): MatchItem[]` — Season format: "2025-2026". Uses getSeasonFromDate to check each match's date. Matches with no date are excluded.

- `computeWinLossStats(matches: MatchItem[]): WinLossStats` — Count wins/losses from matches with known isWin. winPercentage = Math.round((wins / total) * 100) or 0 if no known results.

- `getSeasonFromDate(dateStr: string): string` — French badminton season runs September to August. Month >= 8 (September, 0-indexed) → `${year}-${year+1}`. Month < 8 → `${year-1}-${year}`.

- `getAvailableSeasons(matches: MatchItem[]): string[]` — Collect unique seasons from match dates, sort descending (most recent first).

- `getDisciplineCounts(matches: MatchItem[]): Record<DisciplineFilter, number>` — Count matches per discipline. 'all' = total count. Used for chip badge display.

**1. Create `src/hooks/useMatchHistory.ts`:**

Follow the same pattern as `useDashboardData` hook (Promise.allSettled, cancelled flag, isLoading/isRefreshing split, error classification with i18n keys):

```typescript
export interface MatchHistoryData {
  sections: MatchSection[];          // Grouped and filtered
  allMatches: MatchItem[];           // All matches (unfiltered)
  stats: WinLossStats;              // Filtered stats
  disciplineCounts: Record<DisciplineFilter, number>;
  availableSeasons: string[];
  activeDiscipline: DisciplineFilter;
  activeSeason: string | null;       // null = current/all
  expandedMatchId: string | null;
  isLoading: boolean;
  isRefreshing: boolean;
  error: string | null;
  setDiscipline: (d: DisciplineFilter) => void;
  setSeason: (s: string | null) => void;
  toggleMatchExpand: (id: string) => void;
  refresh: () => Promise<void>;
}
```

Implementation details:
- Import `getResultsByLicence` from `../api/ffbad`
- Import all utility functions from `../utils/matchHistory`
- Import `useSession` from `../auth/context`
- Import `NetworkError, ServerError` from `../api/errors`
- Use `useCallback` for fetchData, `useEffect` for initial load
- State: allMatches (MatchItem[]), activeDiscipline (DisciplineFilter, default 'all'), activeSeason (string | null, default null), expandedMatchId (string | null), isLoading, isRefreshing, error
- fetchData: call getResultsByLicence(licence), on success map Retour array through toFullMatchItem, store in allMatches. Handle errors: NetworkError → 'matchHistory.networkError', others → 'matchHistory.loadError'
- Derived state via useMemo:
  - filteredMatches = filterByDiscipline(allMatches, activeDiscipline) then filterBySeason if activeSeason set
  - sections = groupByTournament(filteredMatches)
  - stats = computeWinLossStats(filteredMatches)
  - disciplineCounts = getDisciplineCounts(allMatches) — counts always from ALL matches (not filtered)
  - availableSeasons = getAvailableSeasons(allMatches)
- toggleMatchExpand: use LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut) before state change for accordion animation. Toggle expandedMatchId (same id = collapse, different = expand new).
- Import Platform, UIManager from react-native. At module level: enable LayoutAnimation on Android if available.
- setDiscipline and setSeason are simple state setters (useCallback)
- refresh calls fetchData(true)
