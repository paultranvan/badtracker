**Create `app/(app)/(tabs)/club.tsx`** — the Club tab screen:

This screen has two modes: leaderboard view (default) and club search.

**Structure:**
- Import `useClubLeaderboard` and `useClubSearch` hooks from Plan 01
- Import `useSession` to get user's own licence (for row highlighting) and user's club ID from profile
- Import `useDebounce` for search, `useTranslation` for i18n
- Use `useDashboardData` or a direct `getPlayerProfile` call to get the current user's club ID on mount. Since `useDashboardData` returns `profile` which has `club` and `nomClub`, use the session's licence to fetch profile via `getPlayerProfile` (or better: use a lightweight approach — store club info from dashboard in a small state). Simplest approach: call `getPlayerProfile(session.licence)` in a useEffect to get club ID, then pass to `useClubLeaderboard`.
- State: `targetClubId` (defaults to user's club), `searchMode` (boolean), `searchQuery` (string)

**Leaderboard view (default):**
- Club header: club name (from `useClubLeaderboard.clubName`) + member count badge (`t('club.members', { count: rankedCount })`)
- FlatList of `LeaderboardEntry` items with `keyExtractor={(item) => item.licence}`
- Each row: position indicator (`#1`, `#2`, `#3`...), player name (`nom prenom`), best rank badge (e.g., "P10")
- Current user's row highlighted with `backgroundColor: '#eff6ff'`, `borderLeftWidth: 3`, `borderLeftColor: '#2563eb'` (matches app's blue accent)
- Tapping a row: `router.push({ pathname: '/player/[licence]', params: { licence: item.licence } })`
- `RefreshControl` with `isRefreshing` and `refresh` from the hook
- Search icon/button in top-right that switches to search mode
- "My Club" button to return to user's own club (visible when viewing another club)
- Loading state: `ActivityIndicator` centered
- Error state: error message + retry button (same pattern as player profile)
- Empty state when user has no club: show `t('club.noClub')` message + "Browse clubs" button that opens search

**Search mode:**
- TextInput at top with `placeholder={t('club.searchPlaceholder')}`
- FlatList of matching clubs from `useClubSearch`
- Each club row shows club name, tapping navigates back to leaderboard mode with that club's ID
- Minimum 3 chars hint (same as player search)
- Back button/cancel to exit search mode

**Styles:** Follow existing patterns — `#f9fafb` for row backgrounds, `#e5e7eb` borders, `#2563eb` for accents, `#111` for primary text, `#6b7280` for secondary text. Row height and padding consistent with search.tsx result rows.

**Create `app/(app)/club/[clubId].tsx`** — stack route for deep navigation from player profiles:

This is a simplified version of the Club tab that receives `clubId` from route params. It does NOT include club search — just the leaderboard view.

- Use `useLocalSearchParams<{ clubId: string }>()` to get the club ID
- Use `useClubLeaderboard(clubId)` to fetch data
- Use `useSession()` to get current user's licence for row highlighting
- Same FlatList + RefreshControl + row layout as the tab screen (extract shared components or duplicate — given the simplicity, duplicating the render logic is fine; keep it self-contained)
- Same row tap navigation to `/player/[licence]`
- Same loading/error/empty states
- No search functionality — this is a targeted view
