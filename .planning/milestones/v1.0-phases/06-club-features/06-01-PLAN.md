**Create `src/utils/clubLeaderboard.ts`:**

Export a `LeaderboardEntry` interface:
```typescript
interface LeaderboardEntry {
  licence: string;
  nom: string;
  prenom: string;
  bestRank: string;  // e.g. "P10", "NC"
  position: number;  // 1-based after sort
}
```

Export `getRankSortIndex(classement: string): number`:
- Find index in `RANK_BOUNDARIES` (from `../utils/rankings`) where `b.rank.toUpperCase() === classement.toUpperCase().trim()`
- Return `RANK_BOUNDARIES.length` if not found (NC/unknown sorts last)

Export `normalizeToLeaderboard(rawItems: unknown[]): LeaderboardEntry[]`:
- Cast each item to `Record<string, unknown>`
- Filter items that have a truthy `Licence` field
- For each item, build a rankings object matching the `PlayerProfile['rankings']` shape using `buildRanking` helper (same pattern as ffbad.ts — extract ClassementSimple/Double/Mixte + CPPHSimple/Double/Mixte)
- Call `getBestRanking(rankings)` from `../utils/rankings` to get best discipline
- Fallback bestRank to `'NC'` if `getBestRanking` returns null
- Sort entries ascending by `getRankSortIndex(a.bestRank) - getRankSortIndex(b.bestRank)`
- Add 1-based `position` after sorting
- Helper: `parseCpph(value)` — same logic as ffbad.ts (parse string/number to number | undefined)
- Helper: `buildRanking(classement, cpph)` — returns `{ classement, cpph }` or undefined if no classement

**Create `src/hooks/useClubLeaderboard.ts`:**

Follow the `useDashboardData` hook pattern exactly (useCallback + useEffect with cancelled flag, isLoading/isRefreshing/error/refresh).

```typescript
export function useClubLeaderboard(clubId: string | null)
```

Returns `{ members: LeaderboardEntry[], clubName: string, rankedCount: number, isLoading, isRefreshing, error, refresh }`.

- When `clubId` changes (and is non-null), fetch via `getClubLeaderboard(clubId)`
- If `Retour` is array: call `normalizeToLeaderboard(Retour)` to get sorted members
- Extract `clubName` from first item's `NomClub` field (or empty string)
- `rankedCount` = count of members where `bestRank !== 'NC'`
- Error handling: classify `NetworkError` vs other, use i18n keys `club.networkError` and `club.loadError`
- If `clubId` is null: set members to empty, no fetch

**Create `src/hooks/useClubSearch.ts`:**

```typescript
export function useClubSearch()
```

Returns `{ clubs: ClubSearchResult[], isLoading, error, search: (query: string) => void }`.

Define `ClubSearchResult` interface: `{ id: string, name: string }`.

- On first call to `search()`, fetch and cache the full club list via `getClubList()` (store in module-level variable to avoid refetching). Also cache in AsyncStorage with key `badtracker_club_list` and 24-hour TTL.
- Once list is loaded, filter client-side: match items where `(item.NomClub ?? item.Nom ?? '').toLowerCase().includes(query.toLowerCase())`
- Only return results when query is 3+ characters (same pattern as player search)
- Normalize each matching item to `{ id: item.ID_Club ?? item.Club ?? '', name: item.NomClub ?? item.Nom ?? '' }`
- Use `useDebounce` hook from `../hooks/useDebounce` with 300ms delay on the search query
- Error handling: set error string on fetch failure, clear on retry
