  <action>
For each hook, apply the same cache-first-then-fetch pattern. Import `cacheGet` and `cacheSet` from `../cache/storage` and `useConnectivity` from `../connectivity/context`.

**Pattern for each hook (useDashboardData as reference):**

1. Add `const { isConnected } = useConnectivity();` at the top of the hook
2. In `fetchData`:
   a. **Cache read phase (before network):** If NOT a refresh (`!isRefresh`), call `cacheGet` with the appropriate key. If cached data exists, set all state from cache. This gives instant UI.
   b. **Connectivity check:** If `!isConnected` after loading cache, set `isLoading` to false and return early (use cached data only). If no cache AND offline, set error to the appropriate network error i18n key.
   c. **Network fetch phase:** Keep existing fetch logic unchanged.
   d. **Cache write phase (after successful fetch):** After successful API response processing, call `cacheSet` with the normalized data. Fire-and-forget (no await needed in the main flow).
   e. **Error handling update:** In catch block, if we already loaded data from cache, silently return instead of showing error. Only show error if no cached data was available.

3. Add a `useEffect` that watches `isConnected` — when it transitions from false to true, trigger a refresh (auto-refresh on connectivity return per user decision). Use a ref to track previous isConnected value.

**Specific cache keys and data shapes:**

- `useDashboardData`: key `dashboard:${licence}`, cache `{ profile, recentMatches, quickStats, allMatches }`
- `useMatchHistory`: key `matches:${licence}`, cache the raw `allMatches: MatchItem[]` array (filters are applied client-side via useMemo so caching the base data is sufficient)
- `useRankingEvolution`: key `ranking:${licence}`, cache the `chartData: ChartData` object
- `useClubLeaderboard`: key `club:${clubId}`, cache `{ members, clubName, rankedCount }`

**Important implementation details:**
- Do NOT cache when `isRefresh` is true for the cache-read phase (pull-to-refresh should always go to network first, then update cache on success)
- DO still write to cache after a successful refresh (pull-to-refresh updates cache)
- The `useConnectivity()` call requires the component/hook to be inside `ConnectivityProvider` (which Plan 01 wired at root level)
- Preserve all existing hook signatures — no breaking changes to return types
  </action>
