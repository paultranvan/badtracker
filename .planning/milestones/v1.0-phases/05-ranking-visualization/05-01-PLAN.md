1. Create `src/utils/rankingChart.ts` with:

   **Types:**
   ```typescript
   export type Discipline = 'simple' | 'double' | 'mixte';

   export interface ChartDataPoint {
     value: number;          // CPPH value (0 for NC)
     label?: string;         // X-axis date label
     date: string;           // ISO date for sorting
     rank: string;           // Current rank at this point
     isMilestone: boolean;   // True if rank changed from previous point
     discipline: Discipline;
   }

   export interface DisciplineChartData {
     discipline: Discipline;
     color: string;
     points: ChartDataPoint[];
     milestones: ChartDataPoint[];  // Subset where isMilestone=true
     visible: boolean;
   }

   export interface ChartData {
     disciplines: DisciplineChartData[];
     dateRange: { start: string; end: string };
     maxValue: number;
     minValue: number;
   }
   ```

   **Constants:**
   ```typescript
   export const DISCIPLINE_COLORS: Record<Discipline, string> = {
     simple: '#2563eb',   // Blue (matches existing app primary)
     double: '#16a34a',   // Green
     mixte: '#d97706',    // Amber
   };
   ```

   **Functions:**

   a) `parseCpphValue(value: string | number | undefined): number` — Parse CPPH from API response (may be string or number). Returns 0 if undefined/NaN. Reuses the pattern from `src/api/ffbad.ts` parseCpph but returns 0 instead of undefined for chart friendliness.

   b) `parseDate(dateStr: string): Date` — Parse date from FFBaD format. Try ISO first, then DD/MM/YYYY (French format), then fallback to new Date(dateStr). Return epoch (1970-01-01) if all parsing fails.

   c) `formatDateLabel(dateStr: string): string` — Format date for X-axis: "Sep '25" style (3-letter month + 2-digit year). Use French month abbreviations.

   d) `detectMilestones(points: ChartDataPoint[]): ChartDataPoint[]` — Filter points where rank changed from previous point. First point is never a milestone. Compare current.rank !== previous.rank (case-insensitive).

   e) `transformEvolutionData(rawData: Array<Record<string, unknown>>): ChartData` — Main transformation function:
      - Group raw items by discipline (check 'Discipline' field, normalize S/D/M to simple/double/mixte using same mapping from Phase 4 matchHistory.ts)
      - For each discipline, create ChartDataPoint[] sorted by date
      - CPPH value from Points or CPPH field (try both), rank from Classement field
      - If a discipline has no data, create it with empty points (will be shown as NC flat line at 0 in chart)
      - Run detectMilestones on each discipline's points
      - Compute dateRange, maxValue, minValue across all disciplines
      - Return ChartData with all three disciplines (even if empty)

   f) `buildFlatLineForNC(dateRange: { start: string; end: string }): ChartDataPoint[]` — Create 2 points at value 0 spanning the full date range for NC disciplines. These get plotted as a flat line at the bottom.

2. Create `src/hooks/useRankingEvolution.ts`:

   Follow the established hook pattern from `useDashboardData` and `useMatchHistory`:
   - Takes `licence: string` parameter
   - Returns `{ chartData: ChartData | null, isLoading, isRefreshing, error, refresh }`
   - Calls `getRankingEvolution(licence)` from ffbad.ts
   - Handles the Retour union (string = error/no data, array = evolution items)
   - Transforms response through `transformEvolutionData`
   - Uses `useCallback` + `useEffect` with cancelled flag cleanup (same as useDashboardData)
   - Error classification: NetworkError -> 'ranking.networkError', others -> 'ranking.loadError'
   - For NC disciplines with no evolution data, uses `buildFlatLineForNC` with the date range from disciplines that DO have data
