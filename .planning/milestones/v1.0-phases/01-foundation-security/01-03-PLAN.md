    **SecureStore helpers (src/auth/storage.ts):**

    Create functions wrapping expo-secure-store for credential management:

    ```typescript
    import * as SecureStore from 'expo-secure-store';

    const CREDENTIALS_KEY = 'ffbad_credentials';
    const REMEMBER_KEY = 'ffbad_remember';
    ```

    `storeCredentials(licence: string, password: string, remember: boolean)`:
    - If remember is true: store credentials as JSON in SecureStore with `keychainAccessible: SecureStore.AFTER_FIRST_UNLOCK` (needed for background auto-login before device unlock)
    - Also store REMEMBER_KEY = 'true' in SecureStore
    - If remember is false: store REMEMBER_KEY = 'false', do NOT store credentials

    `getStoredCredentials(): Promise<{ licence: string; password: string } | null>`:
    - Check REMEMBER_KEY — if not 'true', return null
    - Read CREDENTIALS_KEY from SecureStore
    - Parse JSON, return credentials or null on any error

    `clearCredentials(): Promise<void>`:
    - Delete CREDENTIALS_KEY from SecureStore
    - Delete REMEMBER_KEY from SecureStore
    - NOTE: Per user decision, do NOT clear cached data (matches, rankings) — only clear auth

    `isRememberEnabled(): Promise<boolean>`:
    - Read REMEMBER_KEY, return true if 'true'

    **Session context (src/auth/context.tsx):**

    Create a React Context providing session state to the entire app:

    Interface:
    ```typescript
    interface SessionContextType {
      session: { licence: string; nom: string; prenom: string } | null;
      isLoading: boolean;
      signIn: (licence: string, password: string, remember: boolean) => Promise<void>;
      signOut: () => Promise<void>;
    }
    ```

    `SessionProvider` component:
    1. On mount, check for stored credentials via `getStoredCredentials()`
    2. If credentials found (remember was on):
       - Set `isLoading: true` (splash screen stays visible)
       - Call `setCredentials()` from api/client.ts to inject into axios
       - Call `validateCredentials()` from api/ffbad.ts to verify they still work
       - If valid: set session state with player info, set `isLoading: false`
       - If invalid (AuthError): clear stored credentials, set session null, set `isLoading: false` — user sees login screen
       - If network error: set session from stored credentials anyway (offline tolerance), set `isLoading: false`
       - TIMEOUT: If validation takes >5 seconds, proceed with stored credentials (per pitfall #5 from research — don't block app start)
    3. If no stored credentials: set session null, set `isLoading: false` immediately

    `signIn` function:
    1. Call `validateCredentials(licence, password)` via FFBaD API
    2. If success: call `setCredentials()` to inject into axios client
    3. Call `storeCredentials(licence, password, remember)` to persist if remember is on
    4. Set session state with player info from API response
    5. If failure: throw the error (let the login screen handle display)

    `signOut` function:
    1. Call `clearCredentials()` from storage
    2. Call `setCredentials(null)` to clear axios client state
    3. Set session to null (triggers route guard → login screen)

    Export `useSession()` hook that throws if used outside provider.
