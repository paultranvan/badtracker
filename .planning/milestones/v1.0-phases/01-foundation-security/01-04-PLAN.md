    ```bash
    curl -s "https://apitest.ffbad.org/rest/?AuthJson=%7B%22Login%22:%22test%22,%22Password%22:%22test%22%7D&QueryJson=%7B%22Fonction%22:%22ws_getlicenceinfobylicence%22,%22Param%22:[%22test%22,false]%7D" | jq .
    ```

    Based on actual responses:
    1. Update Zod schemas in `src/api/schemas.ts` to match real field names and types
    2. Add any missing fields discovered in the response
    3. Ensure `.passthrough()` on objects so future API additions don't break validation
    4. Test with French accented names to verify UTF-8 handling

    **Edge case hardening in `src/api/client.ts`:**

    1. HTTPS enforcement check: Add a runtime guard that rejects any request URL not starting with `https://`
       ```typescript
       apiClient.interceptors.request.use((config) => {
         if (config.baseURL && !config.baseURL.startsWith('https://')) {
           throw new Error('HTTPS required: insecure requests are blocked');
         }
         return config;
       });
       ```

    2. UTF-8 response handling: Ensure axios responseType is 'json' and responseEncoding is 'utf8'

    3. Timeout handling: Map axios timeout errors (code 'ECONNABORTED') to NetworkError

    4. Empty response handling: If response.data is null, undefined, or empty string, throw SchemaValidationError

    5. String error responses: FFBaD may return error messages as plain strings in `Retour` field — update schemas to handle `z.union([dataSchema, z.string()])` pattern and check if Retour is a string (indicating error)
  </action>
